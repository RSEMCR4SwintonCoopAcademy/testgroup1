# Template repository for creating individual repositories for each group in the class

This is a template repository to be used for creating individual repositories for class groups at Salford City Academy to do virtual work experience projects that progress in a similar way to how we work in the UoM RSE department.

## Instructions for admins (can be deleted when creating the actual repos)

Add some instructions here as to how to clone and populate class group repositories from this repository.

As each separate group progresses, we will be keeping an eye on what they do, and we can set them further tasks based on what they have done so far in later weeks.

## Instructions for class groups (keep this when creating the actual repos)

Here we will add some instructions for the project.

Essentially what we are looking for is a website describing a company; ideally, a fictitious company that the students come up with themselves, but if they are struggling for ideas then we can make some suggestions.

Some ideas for various sections of the website will be in the "Issues" tab for their particular repository.

The `index.html` file in the `public_html` repository will be the landing point (i.e. the home page) for your website. Start from this page (and add new pages as appropriate).

## Rough outline of work to be done each week

This is a rough outline of what to do

- Session 1: Members of the UoM RSE team introduce ourselves (in-person), we explain what the processes we follow are and provide them with a repository and project with a backlog of github issues, we tell them to set up their "organisation" in terms of the list of "managers" in the briefing document by following the instructions in the issue for deciding who is going to do what job. Depending on technology, this may be all we are able to do that week.

- Session 2: The class groups work on the initial items in the backlog in the first iteration (with one of us on hand to help with processes but not the actual work).

- Session 3: Evaluate in your group what happened in session 2, decide on further work, and carry on with more backlog items. If you need advice from the UoM RSE team then at least one of us will be on hand to help virtually, or we can also work out a way to answer queries in between sessions if needed.

- Session 4: Evaluate what happened in session 3, decide on further work, carry on with some backlog items; consider how you will make a HTML "presentation" based on the website they have created.

- Session 5: They do the presentation to us of the website (in-person?), and we also look at the project boards / repos.

- Session 6: Decision on prizes etc.
